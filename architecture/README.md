# 建築 - システムの骨格

## 日本のシステム思考に基づく設計

### 間（ま）の哲学
- 各コンポーネントの間に適切な余白を設ける
- 過剰な結合を避け、緩やかな連携を重視
- 変化に柔軟に対応できる隙間を残す

### 組物（くみもの）構造
- 小さな部材（エージェント）が互いに支え合う
- 釘や接着剤に頼らず、噛み合わせで成立
- 一部の損傷が全体の崩壊につながらない

### 流れ（ながれ）の設計
- データと制御の流れを水路のように設計
- 淀みなく、しかし急ぎすぎず
- 自然な重力（優先順位）に従って流れる

## 技術的構成

- **島々ネットワーク**: 912リポジトリの分散配置
- **通信水路**: gitコミットを介した非同期メッセージング
- **型の倉庫**: CapDBによるコンパイル済み機能の管理
- **技の言語**: FLUXバイトコードによる統一表現
